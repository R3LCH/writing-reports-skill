# 📝 Навыки для подготовки отчетов

**Превращайте черновые требования, шаблоны и реквизиты в проверенные Pandoc-отчеты, которые AI-агент может сдать без импровизации.**

Writing Reports Skills дает Claude Code, Cursor, Gemini, Codex-style агентам и другим загрузчикам навыков повторяемый процесс подготовки учебных, технических и организационных отчетов. Навыки помогают агенту классифицировать входные файлы, сохранять семантический Markdown-исходник, корректно использовать Pandoc и проверять DOCX/PDF/HTML/Markdown перед завершением.

🌐 Языки: [English](README.md) | Русский | [中文](README.zh.md)

## 🚀 Зачем использовать

Большинство проблем при генерации отчетов возникают не из-за Pandoc. Типичные причины: непонятная роль шаблона, выдуманные реквизиты, титульный лист используется как reference для тела, ломается нумерация страниц, появляется поврежденный текст, а агент сообщает о готовности без проверки файла.

Эти навыки дают агенту рабочий процесс, близкий к production-подходу: уточнить недостающие входные данные, классифицировать файлы, сохранить воспроизводимый исходник, выполнить конвертацию и проверить итоговые артефакты.

## ✨ Что вы получаете

- 🧱 **Надежный Pandoc pipeline** для DOCX, PDF, HTML и Markdown.
- 🗂️ **Дисциплину шаблонов** для титульных листов, reference DOCX, полных примеров, CSL, CSS, библиографий и исходников.
- 🔁 **Воспроизводимые отчеты** на основе семантического Markdown и YAML frontmatter.
- 🌐 **Получение требований по URL** с официальных страниц и PDF: агент загружает источник, извлекает правила, сохраняет доказательства и применяет требования.
- 🔒 **Безопасную работу с реквизитами**: агент не выдумывает ФИО, группу, преподавателя, клиента или организацию.
- 📄 **Корректный сценарий "титульный лист + тело"** без использования cover-only DOCX как `--reference-doc`.
- ✅ **Проверку перед завершением**: файлы, обязательные разделы, цитирования, поля, нумерация, плейсхолдеры, поврежденный текст, пустые страницы, цвета и оформление листингов.
- 📐 **Поддержку ГОСТ-style отчетов**: студенческие отчеты, лабораторные, курсовые, титульные листы, поля, нумерация страниц и листинги исходного кода.

## 🎯 Когда использовать

Репозиторий полезен, если AI-агент должен помочь с:

- лабораторными, курсовыми, студенческими и дипломными отчетами;
- техническими отчетами из Markdown;
- DOCX/PDF-файлами по reference-документу, загруженным требованиям или официальному URL с правилами оформления;
- отчетами с титульными листами, реквизитами, таблицами, рисунками, цитированием и листингами;
- повторяемой подготовкой отчетов в Claude Code, Cursor, Gemini и Codex-like инструментах.

## ⚡ Быстрый старт

### Claude Code Plugin

Установите плагин из Claude Code marketplace после того, как marketplace станет доступен:

```bash
claude plugin install writing-reports-skills@writing-reports-marketplace
```

Для использования без установки плагина скопируйте или объедините `CLAUDE.md` с инструкциями целевого проекта.

### Cursor

Откройте репозиторий в Cursor. Правило `.cursor/rules/writing-reports.mdc` уже добавлено с `alwaysApply: true`.

Для другого проекта скопируйте `.cursor/rules/writing-reports.mdc` в каталог `.cursor/rules/` этого проекта.

### Gemini

Скопируйте или объедините `GEMINI.md` с инструкциями проекта Gemini CLI, либо используйте этот репозиторий как источник папок `skills/`.

### Codex и другие загрузчики навыков

Скопируйте папки из `skills/` в каталог навыков вашего инструмента или укажите репозиторий напрямую, если инструмент поддерживает навыки из репозитория.

## 🧩 Включенные навыки

- `writing-reports-pandoc` - общий процесс создания, оформления, конвертации и обновления отчетов через Pandoc, DOCX, PDF, HTML, Markdown, reference-документы, титульные листы, цитирования и шаблоны.
- `writing-reports-pandoc-gost` - строгий процесс для русскоязычных ГОСТ-style студенческих отчетов, лабораторных, курсовых, DOCX/PDF, титульных листов, нумерации, полей и простых листингов.

## 💬 Примеры запросов агенту

```text
Используй writing-reports-pandoc, чтобы создать DOCX и PDF из report.md, reference.docx и credentials.yaml. Сохрани Markdown-исходник и проверь выходные файлы.
```

```text
Используй writing-reports-pandoc-gost, чтобы подготовить ГОСТ-style лабораторный отчет с титульным листом, нумерацией страниц, Times New Roman 14, интервалом 1.5 и простыми листингами кода.
```

```text
Классифицируй файлы в папке отчета, определи, какой DOCX является только титульным листом, а какой можно использовать как reference для тела, затем собери итоговый отчет.
```

```text
Используй writing-reports-pandoc, чтобы загрузить правила оформления с официального URL, сохранить извлеченные требования в report-guidelines.md, затем создать DOCX и PDF из report.md.
```

## 🌐 Правила оформления по URL

Можно передать агенту официальный URL с методическими указаниями или правилами оформления вместо ручного перечисления всех требований. Навыки инструктируют агента:

1. загрузить URL доступными web/browser-инструментами;
2. извлечь применимые правила: поля, шрифты, интервалы, титульный лист, нумерацию страниц, цитирование, обязательные разделы и форматы сдачи;
3. сохранить URL, дату получения, название страницы/документа, тип контента и извлеченные требования;
4. попросить вставить текст или загрузить файл, если URL закрыт, требует авторизации или недоступен;
5. применить извлеченные правила до использования дефолтов.

Так требования из URL становятся проверяемыми, а не скрытыми в памяти агента.

## 🧪 Примеры файлов

- `examples/credentials.example.yaml` - пример реквизитов с плейсхолдерами.
- `examples/first-page.example.md` - пример исходника титульного листа.
- `examples/report.example.md` - пример семантического Markdown-исходника отчета.

Используйте примеры как безопасную стартовую точку: скопируйте их в приватную папку отчета, замените плейсхолдеры реальными данными там, затем попросите агента собрать отчет из этих файлов. Не храните реальные реквизиты в этом репозитории.

Не коммитьте реальные данные студентов, преподавателей, клиентов, организаций или API. Заменяйте плейсхолдеры только в приватных рабочих копиях.
