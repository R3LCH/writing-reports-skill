# Навыки для подготовки отчетов

Превращайте разрозненные требования к отчету в проверенные Pandoc-выходы, которые AI-агент может уверенно сдать.

Writing Reports Skills дает Claude Code, Cursor, Gemini, Codex-style агентам и другим загрузчикам навыков повторяемый процесс подготовки учебных, технических и организационных отчетов. Вместо того чтобы заставлять агента импровизировать с DOCX/PDF-оформлением, репозиторий учит его классифицировать входные файлы, сохранять семантический Markdown-исходник, корректно использовать Pandoc и проверять результат перед завершением.

Языки: [English](README.md) | Русский | [中文](README.zh.md)

## Зачем это нужно

Большинство ошибок при генерации отчетов возникают не из-за Pandoc. Причины обычно другие: непонятная роль шаблона, выдуманные реквизиты, титульный лист используется как reference-документ для тела, ломается нумерация страниц, появляется поврежденный текст, а агент слишком рано сообщает о готовности.

Эти навыки снижают такие риски. Агент получает четкий процесс: уточнить недостающие данные, разделить роли файлов, сохранить воспроизводимый исходник, выполнить конвертацию и проверить артефакты.

## Что вы получаете

- **Надежный Pandoc workflow** для DOCX, PDF, HTML и Markdown.
- **Дисциплину шаблонов**: титульные листы, reference DOCX, полные примеры, CSL, CSS, библиографии и исходники не смешиваются между собой.
- **Воспроизводимый источник** в виде Markdown/YAML вместо одноразовых ручных правок DOCX.
- **Безопасную работу с реквизитами**: агент не выдумывает ФИО, группу, преподавателя, клиента или организацию.
- **Корректный сценарий "титульный лист + тело"** без использования cover-only DOCX как `--reference-doc`.
- **Проверку перед завершением**: наличие файлов, обязательные разделы, цитирования, поля, нумерация, плейсхолдеры, поврежденный текст, пустые страницы, цвета и оформление листингов.
- **Поддержку ГОСТ-style отчетов**: студенческие отчеты, лабораторные, курсовые, GUAP-style DOCX/PDF, титульные листы, поля, нумерация страниц и листинги исходного кода.

## Когда использовать

Репозиторий полезен, если AI-агент должен помочь с:

- лабораторными, курсовыми, студенческими и дипломными отчетами;
- техническими отчетами из Markdown;
- DOCX/PDF-файлами по reference-документу или требованиям вуза;
- отчетами с титульными листами, реквизитами, таблицами, рисунками, цитированием и листингами;
- повторяемой подготовкой отчетов в Claude Code, Cursor, Gemini и Codex-like инструментах.

## Быстрый старт

### Claude Code Plugin

Установите плагин из Claude Code marketplace после того, как marketplace станет доступен:

```bash
claude plugin install writing-reports-skills@writing-reports-marketplace
```

Для использования без установки плагина скопируйте или объедините `CLAUDE.md` с инструкциями целевого проекта.

### Cursor

Откройте репозиторий в Cursor. Правило `.cursor/rules/writing-reports.mdc` уже добавлено с `alwaysApply: true`.

Для другого проекта скопируйте `.cursor/rules/writing-reports.mdc` в каталог `.cursor/rules/` этого проекта.

### Gemini

Скопируйте или объедините `GEMINI.md` с инструкциями проекта Gemini CLI, либо используйте этот репозиторий как источник папок `skills/`.

### Codex и другие загрузчики навыков

Скопируйте папки из `skills/` в каталог навыков вашего инструмента или укажите репозиторий напрямую, если инструмент поддерживает навыки из репозитория.

## Включенные навыки

- `writing-reports-pandoc` - общий процесс создания, оформления, конвертации и обновления отчетов через Pandoc, DOCX, PDF, HTML, Markdown, reference-документы, титульные листы, цитирования и шаблоны.
- `writing-reports-pandoc-gost` - строгий процесс для русскоязычных ГОСТ-style студенческих отчетов, лабораторных, курсовых, GUAP-style DOCX/PDF, титульных листов, нумерации, полей и простых листингов.

## Примеры запросов агенту

```text
Используй writing-reports-pandoc, чтобы создать DOCX и PDF из report.md, reference.docx и credentials.yaml. Сохрани Markdown-исходник и проверь выходные файлы.
```

```text
Используй writing-reports-pandoc-gost, чтобы подготовить ГОСТ-style лабораторный отчет с титульным листом, нумерацией страниц, Times New Roman 14, интервалом 1.5 и простыми листингами кода.
```

```text
Классифицируй файлы в папке отчета, определи, какой DOCX является только титульным листом, а какой можно использовать как reference для тела, затем собери итоговый отчет.
```

## Примеры файлов

- `examples/credentials.example.yaml` - пример реквизитов с плейсхолдерами.
- `examples/first-page.example.md` - пример исходника титульного листа.
- `examples/report.example.md` - пример семантического Markdown-исходника отчета.

Не коммитьте реальные данные студентов, преподавателей, клиентов, организаций или API. Заменяйте плейсхолдеры только в приватных рабочих копиях.
